# Workspace Instructions

Scope: everything under `workspace/`.

- Default location for AI-created projects: `workspace/<project-name>/`
- Do not create new application projects in the repository root.
- Keep one project per top-level directory inside `workspace/`.
- Treat `workspace/` as local working area; its project contents are git-ignored by default.
- When creating a new project, prefer a short descriptive slug such as `workspace/go-api` or `workspace/web-landing`.
- Keep project-specific docs and artifacts inside the same project directory instead of scattering them across the template repository.
